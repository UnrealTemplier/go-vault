## 🤖 AI-Workflow (Как наполнять базу)
Мы используем терминальные агенты (CLI) для автоматической генерации статей.

1.  Для основных агентов прописан системный промпт в файлах GEMINI.md, CLAUDE.md, QWEN.md
2.  В статье **НЕ ДОЛЖНО** быть дублирующего заголовка ` # Название`, если оно совпадает с именем файла.

**Пример команды для терминала для простого заполнения статьи (это будет учитывать системный промпт, например, в QWEN.md):**
```bash
qwen "Заполни статью Папка/Файл" -y
```

**Пример команды для терминала для создания полного руководства по более гибким правилам заполнения (это модифицирует системный промпт в QWEN.md):**
```bash
qwen "Можешь игнорировать либо гибко следовать ранее определенным правилам. НО соблюдай правила по последнему блоку Связанные темы. Мне нужно полное руководство программиста. Заполни статью 02_Bitrix_Framework/06_Modules/4. Currency" -y
```

## 🛠 Стандарты коммитов (Conventional Commits)
Для чистоты истории Git используй этот формат: `тип(область): описание`
*Пример: `feat(bitrix): add ORM documentation`*

*   **`feat`** — Новая статья/раздел.
*   **`fix`** — Исправление ошибок в статьях.
*   **`docs`** — Изменения в README, PROMPTS или MOC.
*   **`perf`** — Улучшение связей (Графа) в Obsidian.
*   **`chore`** — Рутина (скрипты, настройка структуры).
*   **`refactor`** — Перенос файлов, изменение иерархии.

> **Важно:** Сообщение пишется с маленькой буквы, в повелительном наклонении, без точки в конце.
